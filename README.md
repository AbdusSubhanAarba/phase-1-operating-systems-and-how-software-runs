# phase-1-operating-systems-and-how-software-runs

This repository documents Phase 1 of my structured software engineering learning path, focused on operating systems and how software runs on a computer.

The goal of this phase was to understand what an operating system does, how it manages hardware and software resources, how programs execute, and how core runtime concepts such as processes, threads, memory, file systems, and application crashes work.

This phase builds directly on the hardware and system architecture foundations established in Phase 0.5 and prepares the foundation for networking, programming, and deeper software engineering concepts.

## Topics Covered

- What Is an Operating System
- Windows Fundamentals
- Linux Fundamentals
- macOS Fundamentals
- Kernel Basics
- Processes vs Programs
- Threads
- Multitasking
- Memory Management
- Virtual Memory
- Stack vs Heap
- File Systems
- File Permissions
- Environment Variables
- How Applications Execute
- What Crashes and Freezes Mean

## What This Phase Covers

This phase builds understanding of:

- The role of the operating system in managing hardware and software
- Differences between major operating systems such as Windows, Linux, and macOS
- Kernel responsibilities and system-level control
- The difference between programs and running processes
- How threads allow concurrent execution inside a process
- How multitasking works at the operating system level
- How memory is allocated, managed, and protected
- The purpose of virtual memory and how it extends usable memory space
- The difference between stack memory and heap memory
- How file systems organize and store data
- How file permissions control access to files and directories
- How environment variables influence system and application behavior
- How applications are loaded and executed by the operating system
- What crashes and freezes mean from a system perspective

The objective of this phase is to understand how software actually runs on top of hardware, and how the operating system acts as the layer that manages execution, memory, storage, and system stability.
